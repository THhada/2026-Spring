# 0605 수업 정리

## 1. LLM과 Agent의 분업
* Agent 하나는 LLM 하나
* 역할 예시: LLM(where to(분배), authentifiation, legacy, faq, verboser 등등…)
* 설계: LLM이 분업을 할수록 정교해짐. 설계는 사람이 시행착오를 겪으며 함
* Designed by FDE: Fde- forward deployed engineer(전방(현장) 배치 기술자/인력). 팔란티어가 FDE를 가진 대표적 회사

## 2. Agentic AI
* 개념: 사람이 모든 단계를 지시하지 않아도 스스로 계획을 세움(flow 혹은 loop를 만듦)
* Designed by planner LLM
* 작동 방식: 결과물을 또 다시 LLM(evaluator)를 통해 LLM(planner) 혹은 executor에게 보내는 식으로 loop를 돌림 → 피드백
* 한계: 토큰이 많이 필요, 시간도 많이 필요
* 평가: 교수님은 그래도 사람이 하는게 더 믿을만 하다고 보심

## 3. RAG-검색 증강 생성, Retrieval-Augmented Generation
* 등장 배경: LLM이 똑똑하지 않아서 생겨난 것임. Context Window, Context Length
* 작동 방식: 각 페이지의 chunk들을 vector화 하는 LLM(embedor)
* 만들어진 vector는 embeding vector
* 질문을 하나 하면 각 vector 중에 가장 유사한 vector를 고르면 그게 답일 가능성이 높음
* 이는 cosine similarity와 연관됨

## 4. Pre-training, Fine-Tuning, Human-Feedback
* Pre-training: LLM에 훈련을 계속 시킴(text 등, token(알갱이)A를 넣으면 B가 나오도록 훈련). 그냥 말만 잘함
* Fine-tuning: 질문과 답변을 잘 하도록 바꿔주는 과정. Q-A set의 데이터. 이미 만들어진 거대 모델(pre-trained된)에 회사의 혹은 어떤 데이터를 추가로 학습시켜 모델 자체의 가중치(행렬,weight matrix 혹은 체질)을 바꾸는 기술
* Reinforcement learning by human feedback: 출력되면 안 되는 데이터가 출력되지 않도록 막는 과정
* 과적합(overfitting) ↔ 일반화: 훈련에 쓰인 특정 데이터를 가지고만 적용하면 안됨

## 5. 필수 AI 및 IT 용어
* Token: LLM이 텍스트를 처리하고 생성할 때 사용하는 최소 단위의 알갱이
* API
* Client vs Server
* Cloud Computing: 회사 데이터 센터 공간을 빌려 쓰는 서비스. 아마존 AWS, MS Azure
* SaaS, PaaS, IaaS(?): software, Platform, Infra
* 온프레미스: 그들만의 인터넷(인트라넷같은거) 거기에 gpt 같은 서비스가 들어가는 것. 인터넷이 끊긴 상태에서 데이터 구축
* Full Stack: front-end, back-end를 다 할 수 있는 사람을 Full Stack이라고 부름. 이제 그렇게 희소한 영역은 아님
    * Front-end: 사용자가 직접 보고 클릭하는 화면(Javascript, HTML). 디자이너 같은 직군에게 밀려나는 중.
    * Back-end: (python, Java, Node js). back-end 개발로는 웹페이지를 만들 수 없음
* B2B, B2C
* MVP
* UX/UI
* Legacy: 원래 회사에서 쓰던 시스템
* 도메인
* 정형데이터: 엑셀 등에 들어가는 숫자 데이터
* 비정형데이터: 텍스트 ex)VOC, 고객의 소리
* Human in the loop(HITL) ↔ Agentic AI: 쉽게 말하면 우리가 지금 하는 것들
* 가드레일: 정보 유출이나 욕설 이런 것들 못하게 하는 것. 이것도 하나의 프롬프트
